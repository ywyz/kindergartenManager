# 手动测试文档（账号管理第二阶段）

本文用于你进行发布前人工验收，覆盖以下能力：
- 系统管理员初始化脚本
- 账号创建
- 账号启用/停用
- 管理员重置密码
- 账号筛选与分页
- 权限可见性与越权防护

## 1. 前置条件

1. 已配置环境变量并完成迁移：
```bash
.venv/bin/alembic upgrade head
```
2. 应用可启动：
```bash
.venv/bin/python -m app.main
```
3. 浏览器可访问 `http://localhost:8080`。

## 2. 初始化系统管理员

1. 设置环境变量（示例）：
```bash
export BOOTSTRAP_ADMIN_ENABLED=true
export BOOTSTRAP_ADMIN_TENANT_ID=1
export BOOTSTRAP_ADMIN_USERNAME=sysadmin
export BOOTSTRAP_ADMIN_PASSWORD='StrongPass123!'
# 如连接远程数据库，需显式放开（默认 false）
export BOOTSTRAP_ADMIN_ALLOW_REMOTE=true
```
2. 执行初始化脚本：
```bash
.venv/bin/python -m app.jobs.bootstrap_admin
```
3. 预期结果：
- 首次执行：输出 `ok: created sys_admin ...`
- 再次执行：输出 `skip: sys_admin already exists ...`

## 3. 登录与权限可见性

1. 用 `sysadmin` 登录。
2. 在主页确认“账号管理”入口可见。
3. 用普通教师账号登录。
4. 在主页确认“账号管理”入口不可见。
5. 教师账号直接访问 `/user-admin`。
6. 预期结果：页面提示“无访问权限”。

## 4. 创建账号

1. 进入 `/user-admin`。
2. 输入用户名、初始密码、角色，点击“创建账号”。
3. 预期结果：
- 页面提示“账号创建成功”。
- 列表刷新后能看到新账号。
4. 用同名账号再次创建。
5. 预期结果：提示“用户名已存在”。

## 5. 启用/停用账号

1. 在账号管理列表选择目标账号。
2. 点击“停用账号”。
3. 退出并使用该账号登录。
4. 预期结果：登录失败（用户名或密码错误）。
5. 回到管理员账号，点击“启用账号”。
6. 再次使用该账号登录。
7. 预期结果：登录成功。

## 6. 重置密码

1. 在账号管理页面选择目标账号。
2. 输入新密码，点击“重置密码”。
3. 用旧密码登录该账号。
4. 预期结果：旧密码登录失败。
5. 用新密码登录该账号。
6. 预期结果：登录成功。

## 7. 筛选与分页

1. 使用用户名关键字筛选（如 `teacher`）。
2. 切换角色筛选（teacher / teaching_admin / sys_admin）。
3. 切换每页条数（10/20/50），并使用上一页/下一页。
4. 预期结果：
- 列表只显示匹配项。
- 总条数与页码显示正确。
- 翻页按钮在首页/末页状态正确。

## 8. 安全回归检查

1. 以系统管理员身份尝试停用自己账号。
2. 预期结果：提示“不允许修改自己的启用状态”。
3. 尝试将密码设置为短密码（<8）。
4. 预期结果：提示密码长度不足。
5. 不开启 `BOOTSTRAP_ADMIN_ALLOW_REMOTE` 时执行脚本（连接远程库）。
6. 预期结果：脚本拒绝执行并提示 remote blocked。
