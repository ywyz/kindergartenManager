# PRD：幼儿园教学管理系统（首期）

## 1. 文档目的

本文档定义“幼儿园信息管理系统”中首期交付模块：教学管理-每日活动计划撰写与收集。目标是在可用、稳定、安全的前提下，完成从教案输入、AI拆分、年龄适配、活动生成、Word导出到云端存档的完整闭环。

## 2. 产品范围

### 2.1 首期范围（MVP）

1. 账号登录与基础权限（教师、教研管理员、系统管理员）
2. 学期配置与日期计算（第几周、周几、工作日判定）
3. 教案AI拆分并回填表单
4. 活动过程年龄适配、原文对比
5. 一日活动辅助生成（晨间活动/室内区域活动/户外游戏活动）
6. 按模板导出Word并对差异内容标红
7. 数据保存到云端MySQL并可查询
8. 提示词管理（多版本、可回滚）

### 2.2 二期范围（系统稳定后）

1. 对外REST API
2. 数据库只读视图
3. 服务间 API Key + 签名鉴权

## 3. 角色与权限

1. 教师：创建/编辑/导出本班计划、使用AI功能
2. 教研管理员：审核与管理计划、管理提示词版本
3. 系统管理员：账号管理、系统参数、密钥与审计管理

## 4. 核心业务流程

### 4.1 基础配置流程

1. 用户录入学期开始与结束日期
2. 用户选择目标日期
3. 系统计算并展示：第几周、周几、是否工作日
4. 非工作日仅提示，不阻止继续
5. 用户维护年级、班级、区域内容、户外内容并保存

### 4.2 教案拆分流程

1. 用户提交完整教案文本
2. 点击“连接AI拆分”
3. 调用 OpenAI 兼容接口，返回结构化JSON：

- 活动目标
- 活动准备
- 活动重点
- 活动难点
- 活动过程

4. 自动回填到NiceGUI表单

### 4.3 年龄适配与比对流程

1. 基于年龄段改写“活动过程”
2. 生成原文与改写文差异结果
3. 导出Word时将差异部分标红

### 4.4 一日活动生成流程

输入条件：周次、周几、是否临近节假日、班级区域、户外内容

输出内容：晨间活动、区域游戏、户外游戏等建议内容并回填

### 4.5 导出与存档流程

1. 按Word模板填充内容
2. 差异文本标红
3. 生成导出文件
4. 保存结构化数据到MySQL

## 5. Word模板要求

单表格结构如下：

1. 第( )周（整行）
2. 月 日 周()（整行）
3. 晨间活动：体能大循环/集体游戏/自选游戏 + 重点指导/活动目标/指导要点
4. 晨间谈话：谈话主题 + 问题设计
5. 集体活动：活动主题/活动目标/活动重点/活动难点/活动过程
6. 室内区域活动：游戏区域 + 重点指导/活动目标/指导要点 + 支持策略
7. 户外游戏活动：游戏区域 + 重点指导/活动目标/指导要点 + 支持策略
8. 一日活动反思：空白

## 6. 技术方案（首期）

1. 前后端：Python + NiceGUI
2. 数据库：MySQL（tenant_id + user_id隔离）
3. 鉴权：账号密码 + JWT + RBAC
4. AI：OpenAI兼容Chat Completions
5. 文档：python-docx主方案，docxtpl备选
6. 部署：云服务器 + systemd + Nginx

## 7. 数据与安全

1. AI API Key 入库加密保存，页面脱敏展示
2. 密码采用强哈希
3. 操作审计：AI调用、导出、关键配置变更
4. 节假日API带缓存与降级策略

## 8. 非功能要求

1. 稳定性：AI调用重试与超时控制
2. 可维护性：配置分层、迁移脚本、结构化日志
3. 可恢复性：数据库与导出文件备份
4. 模块化：多文件，清晰分层，后续可独立部署API服务与任务服务
5. 禁止单体巨文件，保持代码清晰可读

## 9. 验收标准

1. 可完成“输入教案->AI拆分->回填->改写->对比->导出->存档”闭环
2. 非工作日给出提示但可继续生成
3. Word导出字段完整，差异文本成功标红
4. 数据能按租户、用户、日期查询
5. 提示词支持多版本与回滚

## 10. 风险与应对

1. AI结构化输出波动：强约束JSON schema + 重试
2. 外部节假日接口不稳定：本地缓存与手工兜底
3. Word样式保真风险：模板锁定 + 小样回归测试
