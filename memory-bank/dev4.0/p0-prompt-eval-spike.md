# dev4.0 P0 — Prompt Eval Contract Spike

## 结果指标

证明 dev4.0 提示词发布前有可测试、可审计的 eval 合同，而不是只靠人工判断。

## 范围

- 在 `packages/prompt` 建立提示词 eval 汇总合同。
- 计算结构通过率、必填字段完整率、可导出性、禁 Markdown、字数范围五类门槛。
- 默认发布阈值均为 100%。
- 未达标时默认阻断发布。
- 仅业务园长可填写风险说明进行风险放行。

## 不做

- P0 不调用真实 AI。
- P0 不建立样例集数据库。
- P0 不实现提示词 UI。
- P0 不实现 champion/challenger 流量分配。

## Gate Tests

```bash
pnpm --filter @kindergarten/prompt typecheck
pnpm test:gate
```

测试覆盖：

- 全部样例通过时允许发布。
- 任一门槛未达标时阻断发布并返回阻断原因。
- 业务园长填写风险说明可放行未达标结果。
- 教师、年级组长、园长、系统管理员不能风险放行。

## 失败边界

- 如果低于阈值仍默认允许发布，测试失败。
- 如果系统管理员默认可以风险放行业务提示词，测试失败。
- 如果阻断结果没有原因，测试失败。
