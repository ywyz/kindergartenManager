# 游戏观察子系统 — 手动测试清单

> **对应开发阶段**：E~I（2026-06-10 完成，等待手动验收）
> **测试前提**：新电脑环境搭建完成、数据库迁移至 head、视觉模型 API Key 已配置

## 环境搭建（新电脑）

```bash
# 1. 克隆 / 拉取仓库
git clone git@github.com:ywyz/kindergartenManager.git
cd kindergartenManager
git checkout dev3.0

# 2. 创建虚拟环境并安装依赖
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# 3. 复制并填写环境变量
cp .env.example .env
# 填写：DATABASE_URL / ENCRYPTION_KEY / JWT_SECRET / HOLIDAY_API_URL
# 注意：ENCRYPTION_KEY 和 JWT_SECRET 必须与生产/云端一致，否则已加密 Key 无法解密

# 4. 执行数据库迁移（新增 export_records.observation_id 列）
.venv/bin/alembic upgrade head

# 5. 验证自动化测试（342 passed, 0 failed）
.venv/bin/pytest tests/ -k "not test_word_exporter" -q
.venv/bin/pytest tests/test_word_exporter.py -q

# 6. 启动应用
.venv/bin/python -m app.main
```

## M-CORE：服务层手动测试（Phase F）

> 可在应用启动后通过页面操作验证，或用 Python 脚本调用服务层函数。

### F-1 邀请码生成与注册闭环

- [ ] 以 `sys_admin` 身份登录 → 进入「账号管理」→「邀请码管理」区块
- [ ] 点击「生成邀请码」→ 列表中出现新邀请码（格式：12位大写字母数字）
- [ ] 复制邀请码，访问 `/register` 页面（无需登录可直接访问）
- [ ] 填写邀请码、用户名、姓名、密码 → 点击「注册」→ 提示「等待管理员审核」
- [ ] 尝试用注册的账号登录 → 提示「用户名或密码错误」（未审核，is_active=False）

### F-2 待审核用户审核

- [ ] 以 `sys_admin` 身份登录 → 进入「账号管理」→「待审核用户」区块
- [ ] 找到刚注册的用户 → 点击「审核通过」
- [ ] 再次用该账号登录 → 成功登录进入仪表盘

### F-3 邀请码停用测试

- [ ] 进入「邀请码管理」→ 点击某邀请码的「停用」
- [ ] 用已停用邀请码访问 `/register` 注册 → 提示「邀请码无效或已停用」
- [ ] 再次点击「启用」→ 恢复有效

## Phase G：Word 导出手动验证

> 需要先完成 F-1/F-2，有一个可用账号并配置好视觉模型 Key

### G-1 观察导出文档验证

- [ ] 进入「游戏观察记录」页面 → 填写表单（日期/大环境选「户外」/游戏区域/人数等）
- [ ] 上传 1~3 张游戏照片
- [ ] 点击「生成观察记录」→ 等待 AI 返回（需配置视觉模型 Key）
- [ ] 确认 4 段内容（观察目标/记录/分析/策略）回填到文本框
- [ ] 点击「保存」→ 提示保存成功（含记录 ID）
- [ ] 点击「导出 Word」→ 浏览器下载 `.docx` 文件
- [ ] **打开导出文件**，验证以下内容：
  - [ ] 标题含「户外」（不含 `xx`）
  - [ ] 班级/日期/起止时间/成人数/儿童数/观察者 填入正确
  - [ ] 幼儿姓名/年龄 填入正确
  - [ ] 观察环境（游戏区域）正确
  - [ ] 观察目标/记录/分析/策略 与页面一致
  - [ ] 模板示例文字「王鹤宁、侯舒妍」不残留
  - [ ] 图片在「观察记录」格内文字下方横向并排
  - [ ] 中文字体显示正常（无乱码，宋体）

### G-2 导出记录验证（数据库）

- [ ] 导出后，在 MySQL 中执行：
  ```sql
  SELECT * FROM export_records ORDER BY id DESC LIMIT 1;
  ```
- [ ] 确认 `observation_id` 字段有值（与页面显示的记录 ID 一致）

## 已知注意事项

1. **视觉模型 Key**：测试 H-2（生成观察记录）前必须在设置页配置好视觉模型
2. **图片格式**：上传时选择 JPEG 或 PNG，其他格式可能触发压缩失败提示
3. **邀请码 tenant_id**：邀请码绑定 tenant_id，注册用户的 tenant_id 来自邀请码——需确认 sys_admin 生成邀请码时 tenant_id 与当前登录 tenant_id 一致（首期固定 tenant_id=1）
4. **Alembic head**：新电脑需执行 `alembic upgrade head` 确保 `export_records.observation_id` 列已创建
5. **日志**：若出现异常，查看终端 JSON 日志定位问题
