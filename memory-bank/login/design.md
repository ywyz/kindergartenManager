# 登录系统 — 设计文档

> 配套：[dev-plan.md](dev-plan.md)、[test-plan.md](test-plan.md)、[progress.md](progress.md)。

## 1. 目标与范围

dev3.4 恢复 NiceGUI 登录系统，替换 v3.0.1 单用户直通模式。系统支持首次管理员初始化、用户登录、用户注册待审核、管理员用户管理、管理员重置密码、个人资料与改密，并让业务数据重新按当前登录用户隔离。

本期范围：
- `/` 登录页、`/setup-admin` 首次管理员初始化页、`/register` 注册页。
- `/profile` 个人资料与改密，`/user-admin` 管理员用户管理。
- 所有业务页面使用 JWT 登录态解析出的 `tenant_id/user_id/role`。
- 普通用户可配置自己的 text/vision AI Key；系统设置仅系统管理员可用；提示词管理开放给教研管理员与系统管理员。
- Windows / Linux 安装包可尝试初始化管理员，首次启动页兜底。

## 2. 关键决策

| # | 主题 | 决策 |
|---|------|------|
| 1 | 初始化 | 安装器初始化 + `/setup-admin` 首次启动兜底。 |
| 2 | 注册 | 公开注册只创建待审核教师账号，不自动成为管理员。 |
| 3 | 重置密码 | 本期仅支持管理员重置用户密码；不做邮箱/短信找回。 |
| 4 | 数据范围 | 业务历史按当前登录用户隔离，管理员本期不跨用户管理业务数据。 |
| 5 | AI Key | 每个用户独立配置 text/vision Key。 |
| 6 | 数据库 | 不新增表字段，复用 `user` 与 `ai_api_key`。 |

## 3. 角色权限

| 角色 | 页面能力 |
|------|----------|
| `teacher` | 教学业务页面、个人 AI 配置、个人资料与改密。 |
| `teaching_admin` | 教学业务页面、个人 AI 配置、提示词管理、个人资料与改密。 |
| `sys_admin` | 全部教学业务页面、个人 AI 配置、系统设置、提示词管理、用户管理、个人资料与改密。 |

## 4. 流程

### 首次初始化

1. 系统启动不再自动创建默认 `admin`。
2. 若没有 active `sys_admin`，访问登录页时提示进入 `/setup-admin`。
3. 创建管理员时校验用户名、显示名、密码长度与重复用户名。
4. 初始化成功后可直接登录或自动进入首页。

### 注册与审核

1. 用户在 `/register` 填写用户名、显示名、密码。
2. 系统存在 active `sys_admin` 时创建 `is_active=False`、`role=teacher` 的待审核账号。
3. 系统管理员在 `/user-admin` 审核通过后，用户才能登录。

### 登录态

1. 登录成功后 JWT 写入 `app.storage.user["token"]`。
2. 页面 helper 解码 token，并查询数据库确认账号仍存在、启用、角色未变化。
3. token 无效、过期或账号停用时清理 storage 并跳转登录页。

## 5. 验收标准

1. 空库可通过 `/setup-admin` 创建系统管理员。
2. 管理员可登录、退出、创建用户、审核用户、启停用户、重置密码。
3. 注册用户默认待审核，审核前不能登录，审核后可以登录。
4. 普通用户进入业务页面后数据写入当前用户 ID。
5. 普通用户能配置个人 AI Key；系统设置只对系统管理员开放。
6. AI 出错时展示可理解、可行动的错误提示。
