## Plan: 幼儿园教学管理首期PRD规划

目标是先聚焦“教学管理-每日活动计划撰写与收集”这一最小可落地子系统，采用可扩展架构预留全园信息化能力。推荐首期以 Python 服务端 + NiceGUI 前端一体化 + 云端 MySQL（租户+用户隔离）+ OpenAI 兼容接口为核心，先稳定实现“计划拆分、AI生成、Word导出、云端存档、提示词管理”闭环，再扩展 Typst 备用链路与全园模块。

**Steps**

1. Phase 1 需求冻结与边界定义
2. 明确首期只交付“教学管理-每日活动计划”业务闭环；账号体系按教师/教研等教学角色优先，其他角色仅做接口预留。
3. 冻结关键规则：学期起止时间、日期选择、第几周/周几/工作日判定、非工作日仅提示可继续。
4. 定义节假日来源为中国法定节假日 API，并保留本地缓存与失败降级策略。
5. Phase 2 信息架构与数据模型设计（依赖 Phase 1）
6. 设计多租户模型：tenant_id + user_id 作为业务数据隔离主键域；覆盖学期配置、班级配置、提示词版本、教案记录、导出记录、AI调用审计。
7. 设计账号与权限：账号密码登录、JWT、RBAC（教师、教研管理员、系统管理员），并预留未来模块权限位。
8. 设计敏感信息策略：AI API Key 入库加密保存（应用层加密 + 密钥轮换预案 + 脱敏展示）。
9. Phase 3 功能流程设计（依赖 Phase 2）
10. 流程 A 基础配置：用户录入学期起止时间、年级、班级、区域内容、户外内容并保存；选择具体日期后实时计算教学周信息。
11. 流程 B 教案拆分：用户提交完整教案文本，点击“连接AI拆分”，返回 JSON（活动目标、活动准备、活动重点、活动难点、活动过程）并回填表单。
12. 流程 C 年龄适配改写：对“活动过程”进行年龄段适配，保留原文与改写文比对结果，作为导出时红字标注依据。
13. 流程 D 一日活动辅助生成：结合第几周、周几、是否临近节假日、班级区域和户外信息，生成晨间活动、区域活动、户外活动建议并回填。
14. 流程 E 提示词管理：提供独立表单管理系统提示词，支持多版本保存、启停与回滚，并绑定不同 AI 任务模板。
15. 流程 F 导出与存档：支持按给定 Word 模板导出；差异内容（年龄适配后与原文不同部分）标红；同时写入云端 MySQL 供检索。
16. Phase 4 文档模板与导出技术评估（可与 Phase 3 并行）
17. 方案优先级：Word 模板直填为主，Typst 为低优先备用。
18. 对比 python-docx 与 docxtpl 在“复杂表格定位、样式保真、局部红字标注、中文字体兼容”上的可行性，输出结论并冻结实现方案。
19. 评估 MarkItDown 在 docx->markdown 仅用于分析辅助，不进入主导出链路，避免反向格式失真风险。
20. Phase 5 非功能设计与交付规划（依赖 Phase 3）
21. 安全：密码强哈希、JWT 过期与刷新、API Key 加密、最小权限、关键操作审计日志。
22. 可靠性：AI 调用超时重试、幂等保存、数据库连接池、节假日 API 降级策略。
23. 可运维：云服务器 systemd 部署、环境变量分层、日志采集、备份与恢复（满足“所有数据备份导出”总目标的首期要求）。
24. 验收与里程碑：M1 原型联调、M2 业务闭环、M3 稳定性与安全加固、M4 上线试运行。
25. Phase 6 二期扩展（M4之后）
26. 对外集成：在系统稳定可用后提供活动计划 REST API（实时读取）与数据库只读视图两种消费方式。
27. 对外接口治理：定义 API 版本策略（v1 起步）、字段稳定性分级、幂等与分页规范、错误码规范、调用审计。
28. 对外鉴权：服务间 API Key + 签名校验。

**Verification**

1. PRD 完整性检查：覆盖业务目标、范围边界、角色权限、流程、数据模型、接口、安全、部署、测试、里程碑。
2. 需求对齐检查：逐条映射你提供的教学管理需求与 Word 模板结构，确保无遗漏。
3. 风险检查：确认“Word 保真导出”“AI 输出结构化稳定性”“多租户隔离”“密钥安全”均有落地策略；对外调用兼容性移入二期评审。
4. 决策留痕检查：所有未冻结项在 PRD 中给出推荐方案、备选方案与决策触发条件。

**Decisions**

- 已确认：PRD 全中文；OpenAI 兼容接口按 Chat Completions；节假日来源使用中国法定节假日 API；非工作日仅提示可继续。
- 已确认：首期聚焦教学管理并预留扩展；云端 MySQL 按 tenant_id + user_id 隔离；AI Key 入库加密；提示词多版本可回滚。
- 已确认：为后续系统升级预留活动计划对外调用能力；该项不进入首期主计划，待系统上线稳定后进入二期实施（REST API + 数据库只读视图，实时读取，服务间 API Key + 签名）。
- 已确认：部署目标为云服务器直装（systemd）；鉴权为账号密码 + JWT + RBAC。
- 已确认：Word 导出主链路采用 python-docx，docxtpl 作为备选方案。

**Further Considerations**

1. Word 导出引擎建议优先做小样验证：Option A python-docx（灵活标红强）/ Option B docxtpl（模板维护友好）/ Option C 双引擎（成本高）。
2. 节假日 API 需要本地缓存和人工覆盖入口，避免第三方接口波动导致教学计划生成异常。
3. Typst 仅保留为长期备用方案，不进入首期关键路径。
