# 周计划子系统总体目标
1. 利用nicegui框架搭建一款周计划生成工具, 核心功能包括
    - 读取数据库中的学期、年级、班级、活动内容等信息
    - 输入还缺少的活动信息
    - 调用AI接口生成完整的周计划里的内容
    - 计划保存与历史回顾
2. word模板读取和导出,便于教师修改和打印

# 周计划子系统设计文档
## 1. 技术选型
- 前端: NiceGUI (基于 Vue3 + Tailwind CSS)
- 后端: Python 3.10+ (FastAPI 内核)
- 数据库: MySQL (PyMySQL 驱动)
- AI 接口: OpenAI 兼容接口 (openai SDK)
- 导出: python-docx 填充 `templates/weekplan.docx`
- 加密: Fernet (cryptography),用于 AI Key 落库前加密
- 节假日: `chinesecalendar`(支持 2004–2026,含调休)
## 2. 功能模块
    我将在此简述我的操作.
    1. 在系统设置中,我会先设置学期、年级、班级基础信息,这些信息会被保存到数据库中以供后续使用.不过之前在一日计划系统中我可能已经保存国这些数据了,所以在周计划系统中我会先读取这些数据,如果已经存在就不再重复输入.
    2. 设置中我还会输入班级教师的姓名和保育员的姓名,这些细细也需要保存.
    3. 在周计划生成界面,我需要选择一周的日期,系统需要自动识别这周的周次,星期,相关日期等信息.
    4. 系统需要查询mysql数据库中本周日期中是否有节假日,如果有,节假日当天的晨间谈话为空,集体活动显示为xx节放假.
    5.我会输入这周的活动主题,系统需要查询本周相关日期有无备课内容,如果有内容,需要放入到周计划的相关表格中,如果没有,集体活动表格留空,等待我输入,其余表格由AI生成.
    6. 需要提供提示词修改的界面,以便我根据实际情况调整AI生成的内容.
    7.最后我会将生成的周计划导出为word文档,以便打印和修改.导出的文档需要填充到`templates/weekplan.docx`模板中.
